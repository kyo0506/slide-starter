# plan.md — 3層分離アーキテクチャ設計書


## 設計思想

HTMLスライドシステムを **Engine / Theme / Content** の3層に分離し、
それぞれの責務を明確にすることで「一度作ったら全デッキに効く」仕組みを実現する。


## 3層の責務

### Engine層（engine/）— 触らない

- **slide.css**: 16:9アスペクト比ロック、ナビゲーションUI、PDF出力用print CSS
- **slide.js**: キーボード操作（←→Space F）、ビューポートスケーリング、PDF書き出しロジック、スライドカウンター
- 全デッキ共通のロジックをここに集約。デッキごとの差異は一切持たない
- **編集禁止の理由**: ここを変えると全既存デッキに影響が波及するため

### Theme層（theme/）— カスタマイズのハブ

- **sample.css**: CSS変数（カラー・フォント・余白・アクセント色）とスライドタイプの定義
- `reference/` の参考画像から抽出したデザイン原則を、CSS変数として実装する場所
- 新しいスライドタイプやユーティリティクラスの追加もここで行う
- **変更時の注意**: 変更したら `decks/01_sample/index.html` で全パターンを目視確認すること

### Content層（decks/）— 自由に書く

- デッキごとにフォルダを分け、`index.html` にスライドを記述
- Theme層のクラスを使って構成するため、デッキ側でのCSS記述は最小限にする
- デッキ固有のスタイル調整が必要な場合は `<style>` ブロックで対応（Theme層は汚さない）


## 拡張手順

### 新しいスライドタイプを追加する場合

1. `theme/sample.css` にクラスとスタイルを追加
2. `CLAUDE.md` のスライドタイプ一覧に追記
3. `decks/01_sample/index.html` にサンプルスライドを追加して動作確認

### 新しいユーティリティクラスを追加する場合

1. `theme/sample.css` にクラスを追加
2. `CLAUDE.md` のユーティリティクラス一覧に追記
3. 必要に応じて `decks/01_sample/index.html` にサンプルを追加

### デザインテイストを変更する場合

1. `reference/` に新しい参考画像を追加
2. Claude Codeに参考画像を分析させ、`design-guidelines.md` を更新
3. `theme/sample.css` のCSS変数を更新
4. `decks/01_sample/index.html` で全パターンを確認
5. ブラウザで見ながら微調整を繰り返す


## ファイルパス規約

- engine/theme へのパスは `../../engine/` `../../theme/` で統一（decks直下の2階層目から参照）
- 共有アセットは `../../shared/` で参照
- デッキ固有アセットは `assets/` （デッキフォルダ内の相対パス）
